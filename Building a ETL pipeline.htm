Building a ETL pipeline

Anthony Inthavong
Dec 17, 2020 · 4 min read

using Python, Pandas, and SQLite

What is a ETL data pipeline?

An ETL (Data Extraction, Transformation, Loading) pipeline is a set of processes used to Extract, Transform, and Load data from a source to a target.

The source of the data can be from one or many sources, such as from an API call, CSV files, information within a database, and many more.

We take these sources of information then transform it in a way where it can be used immediately by another client, user or developer within some target storage.

In practice, an ETL pipeline is run infrequently. Generally there are large amounts of information that is unfiltered and difficult to process and clean, often taking large amounts of time and resources to transform.

Why do we need an ETL pipeline?

Imagine we had a collection of data that we wanted to use. But the problem is this data is uncleaned, missing information, and inconsistent. One solution is to have a program clean and transform this information such that

  1. There is no missing information
  2. Data is consistent
  3. Data is fast to load into another program

There is so much raw, unfiltered data in today’s industry with smart devices, online communities, and E-Commerce. But most of it is wasted because it is to hard to interpret because of how messy it is. ETL pipelines exist to combat this by automating data collection and transformation so it can be used by analysts for business insights.

The ETL pipeline

There are many different tools and technologies we can used to build our data pipeline. This is completely dependent on how our data is collected. In our example we will be collecting raw data from John Hopkins University’s GitHub. Specifically we will be looking at the daily reports. The data spans from January 22, 2020 to December 16, 2020 as of the writing of this article.

Our tech stack will include Python using the libraries Requests and Pandas. We will also be using SQLite to store this information.

1. Extract

Before we can do any sort of data transformation, we need to have data! The data will be collected from John Hopkins University’s Github. We are specifically interested in the daily reports. From Github’s API documentation, we can send a GET request to this endpoint to receive a JSON response back telling us the urls for each csv file so we can read directly.

  • OWNER is owner of GitHub repository
  • REPO is name of repository
  • PATH is specific path to our folder we want to examine relative to REPO
  • URL is api endpoint used to send GET request

Using Python’s Requests library we will interpret the JSON response and examine the information we need. For clarity, the image below is the response returned using a program called Postman. A similar response will be returned using Python’s Request library.
